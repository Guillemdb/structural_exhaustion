## Exact local branch split

### A. Attachment mismatch

Input: the first exact coordinate on which the two retained attachment
predicates differ.

The bare implication “mismatch implies target cycle” is false: an induced
path with pendant leaves at different positions is a tree and has unequal
attachment rows but no cycle.  Therefore the mismatch must first be refined by
token class and by the semantic role of the mismatched coordinate.

Required lemmas:

1. `attachmentMismatch_origin`: project the first coordinate back to the exact
   selected pair, token class, port position, Boolean side, and blocker role.
2. `attachmentMismatch_classify`: a finite token-class-specific table whose
   outputs are a concrete target return, an exact separating-context payload,
   a certified compression proposal, or a typed unmatched-coordinate residual.
3. For every proposed target exit, a literal walk/cycle construction and length
   proof; inequality of rows alone is insufficient.
4. For every CT3 exit, exact same-interface pieces and a concrete response
   coordinate/context.  The context may be proof-carried; it must not be found
   by enumerating ambient completions.

Both-sides predicate: “the mismatched coordinate completes one of the declared
local target returns.”  The positive side gives C1.  The negative side retains
the first failed return field and routes to the token-class table.  It is
measurable from the stored coordinate.  CT10 is the local classifier; CT3 is a
consumer only after its complete trigger tuple is constructed.

### B. Parallel or prefix germs

Input: literal left-prefix or right-prefix equality from the ordered-BFS path
comparison, together with the two original pair supports and attachment rows.

Prefix geometry is not external-response equivalence.  The missing bridge is
an exact pair of boundaried pieces sharing one interface.

Required lemmas:

1. `prefix_common_interface`: cut the longer retained support at the shorter
   terminal and construct two explicit packed pieces with one boundary map.
2. `prefix_reconstruction`: prove that gluing either piece to the retained
   outside complement reconstructs the corresponding original support.
3. `prefix_response_split`: inspect only the declared finite response
   coordinates.  Return a concrete distinguishing coordinate, or equality of
   the complete response vector.
4. `prefix_distinguishing_to_ct3`: package the pieces, interface, and concrete
   coordinate as a CT3 trigger.
5. `prefix_neutral_reduction`: on complete response equality, construct a
   strictly smaller admissible representative and prove the baseline and target
   response obligations needed by minimality.  The strict-rank proof may not be
   replaced by path-length inequality alone.

Both-sides predicate: equality of the finite, proved-complete response vector.
The negative side gives CT3 distinguishing data.  The positive side pays C2
only after the representative and rank decrease are constructed.  Until
response completeness is proved, this predicate is not admitted.

### C. Cubic separator

Input: the exact root or after-edge three-incidence separator and its literal
four-vertex cubic-star decomposition.

The cubic claw supplies an exact interface and reconstruction theorem, but it
does not itself supply a cycle or target-preserving switch.

Required lemmas:

1. The existing `CubicStarDecomposition` gives the three-boundary piece,
   outside complement, graph reconstruction, internal baseline, and linear
   construction budget.
2. `cubic_switch_coordinates`: define the manuscript's finite switch
   representatives on that exact boundary.
3. `cubic_switch_context_generator_complete`: prove that the declared finite
   response coordinates decide every target-relevant compatible context used
   by the switch.  This is the substantive CT3/CT7 equivalence obligation.
4. `cubic_switch_classify`: direct return (C1), distinguishing response (typed
   CT3 payload), neutral smaller switch (C2), or a precisely typed larger-support
   residual for the existing delocalization consumer.
5. `cubic_switch_persistence`: transport the token class, pair provenance,
   attachment rows, and excluded exits to every output.

Both-sides predicate: “the finite switch response table is neutral.”  A failed
row supplies a concrete CT3 coordinate.  A neutral row pays only through a
proved smaller switch.  Geometry alone does not discharge either side.

### D. High separator

Input: the exact root/after-edge incidence, degree at least four, the recovered
declared ports, their open/triangular types, and the finite fan-compatibility
field table.

This branch admits a useful first refinement before any new semantic theorem:
shared shoulders always give a literal four-cycle; an endpoint in the opposite
shoulders gives a four-cycle when either incident port is triangular.  The
remaining open--open endpoint failures and compatible pairs must remain typed
residuals.

Required lemmas:

1. `detailed_separator_normalization`: preserve the exact root/after-edge
   incidences when normalizing cubic/high branches.  The current coarse
   projection retains only the degree lower bound and cannot feed the port
   table.
2. `high_pair_semantic_split`: from the exact two-port table and port
   distinctness, return a literal four-cycle or one of:
   compatible pair, first open--open endpoint failure, second open--open
   endpoint failure.  This split is finite and total.
3. `high_pair_square_sound`: every square constructor yields C1 for target
   length four.
4. `high_compatible_origin`: prove that the two recovered ports coincide with
   the canonical surplus incidences required by the Type-B entry.  This field
   is not currently available.
5. `high_compatible_assignment`: construct the full fan-window profile,
   remainder membership, four shoulder assignments, safety facts, and marked
   provenance required by `FanClosedPort`/Type-B consumers.
6. `high_open_endpoint_failure`: classify the open--open failed endpoint field
   by its exact chord/return arithmetic; route it to a literal target return,
   concrete response defect, or a further typed finite residual.

Both-sides predicate: exact `FanCompatible` for the recovered distinct pair.
Failure first passes through the square split above; unresolved open--open
failures remain finite local data.  Success is not yet Type B: it becomes a
Type-B payload only after origin and full assignment are proved.
