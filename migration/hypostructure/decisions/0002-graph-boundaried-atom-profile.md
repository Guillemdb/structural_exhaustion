# HYP-0002: Graph-owned boundaried atom profiles

Status: implemented

Date: 2026-07-21

Matrix rows: `graph.boundaried-atom-profile`, EG node 11

## Missing Use Case

The Graph layer can represent a finite boundary, a boundary piece, an outside
context, exact gluing, and an owned decomposition. It cannot yet express the
paper's proper connected atom of a minimal graph context, derive its uncapped
boundary-degree profile, enforce the immutable degree-profile fibre used by
target-complete identification, or register that complete result on an active
accumulated branch. Consequently EG node 11 cannot remain a thin framework
invocation.

## Ownership

The capability belongs to Graph. Boundary degree, graph connectedness,
locally owned edge count, graph lexicographic progress, and exact
reconstruction through `OwnedDecomposition` are graph concepts. Core owns the
generic `Residual.Focus` execution and ledger extension but must not acquire
graph vocabulary. CT3 consumes boundaried representatives later; constructing
and profiling an atom is not a CT3 search or outcome.

## Public Author Inputs

- The literal predecessor stage.
- A typed active query for the inherited `Core.MinimalCounterexampleContext`.
- A caller-supplied exact owned decomposition, connectedness proof, and
  intrinsic paper-properness proof when the universal family is evaluated.

The caller does not supply a boundary-degree function, profile equality,
strict-progress certificate, profile-fibre rejection theorem, work claim,
successor payload, route, or ledger extension.

## Framework Outputs

- The exact boundary degree of every labelled interface vertex.
- The uncapped boundary-degree profile of a supplied proper atom.
- A proof that the registered profile is the framework computation.
- The generic rejection of target-complete identifications across distinct
  uncapped profile fibres.
- An exact zero-check count for symbolic profile derivation.
- The complete focused executor count, equal to the inherited selector count
  on both active and inactive outcomes, and its polynomial work envelope.
- Complete proof-relevant metadata containing the actual focused context query,
  author boundary, generated outputs, generic theorems, and work budget.
- One `Core.Residual.Focus` successor containing the complete Graph-owned
  registration.
- A typed query for the newest registration while preserving every
  predecessor query.

## Residual Branches

Node 11 is data registration, not a dichotomy. There is one active successor
and no complementary mathematical residual. Inactive siblings already stored
by the inherited focus remain inactive and receive no fabricated payload.

## Both-Sides Test

The first consumer is EG node 11. The transfer fixture is a second finite
graph with an explicit owned decomposition and computed boundary profile.
There is no honest PDE interpretation of vertex degree or graph
connectedness; reuse is demonstrated by the graph-generic fixture rather than
by introducing graph vocabulary into PDE or Core.

## Fixtures

- Exact boundary-degree and profile values on a finite decomposition.
- Intrinsic properness and connectedness of the supplied atom, with strict
  lexicographic progress derived by Graph.
- Literal predecessor retention by the focused executor.
- Typed retrieval of the generated registration and profile family.
- Rejection of a supplied cross-profile target-complete identification.
- Exact local derivation work, exact total focused-execution work, equality
  between the stored certificate and counted execution, and polynomial bounds.
- One recorded focused query and an empty manual-obligation list.
- Gluing and reconstruction with a context-owned boundary--boundary edge, as
  required by the original union-of-edges definition.
- `#print axioms` audit of public computations and executor.
- Production import-firewall and whitespace checks.

## Compatibility Impact

This adds an independent Graph API and does not import or alias the legacy
boundaried-gluing implementation. Legacy behavior may later be compared only
inside the parity package. The capability adds no trust input and does not
change the unfrozen legacy baseline.

The unrestricted boundary-edge overlap represented by general outside
contexts is discharged at replacement time by Graph's normalized replacement
contract when the predecessor supplies normalized atoms. It does not weaken
this profile registration, and Node 11 remains the broad proper-atom profile
registration rather than a node-local replacement filter.
