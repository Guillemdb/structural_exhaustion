# Maintain the Hypostructure Migration

Track parity by cohesive registered DAG block and final `problemDefinition`, not by compilation count or facade count. Keep source presence, executable strategy coverage, mathematical closure, work, trust, and publication status separate.

Freeze the legacy proof as semantic authority without importing it into production. Compare normalized strategy inputs, exhaustive outputs, and terminal mathematics. Require the application to contain only its problem, DAG, and `ProblemDeclaration.ofDag`; record status from Core's report.

Enforce the import firewall against legacy and manuscript-node dependencies. Permit cutover only when the strategy-native declaration kernel-checks the same theorem with reviewed trust and work evidence.
