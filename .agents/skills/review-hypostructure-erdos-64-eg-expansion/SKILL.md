# Review the Hypostructure Erdős 64 Expansion

Clear defects in the existing touched dependency cone. Do not implement the next white/frontier node. Preserve useful partial code, but demote unsupported status immediately.

Read completely:

- [`../implement-next-hypostructure-erdos-64-eg-ct/references/mandatory-node-template.md`](../implement-next-hypostructure-erdos-64-eg-ct/references/mandatory-node-template.md);

- the exact topology, definitions, and proof in the immutable mathematical authority `original_erdos_64_proof.tex`;
- the corresponding kernel-checked legacy node and framework declarations as implementation/parity evidence;
- the selected CT skill and its live `Spec` through `Automation` layers;
- the implicated manuscript diagram/proof and `migration/hypostructure/eg-node-matrix.csv` rows;
- every direct new predecessor and reviewed facade;
- every consumed Core, Routes, Graph declaration and independent fixture; and
- any parity module, trust entry, and Hypostructure-native evidence cited by the rows.

Inspect the dirty worktree and preserve unrelated edits. Read frozen legacy
code only after reconstructing the obligation from the original manuscript;
use it as checked implementation and parity evidence, never as a production
import, independent mathematical authority, or automatic status promotion.

Never edit `original_erdos_64_proof.tex`. The living manuscript is a
non-binding editorial cross-check and cannot change any reviewed obligation.

## Reconstruct topology and status independently

1. Rebuild every incoming and outgoing edge from `original_erdos_64_proof.tex`, including joins, feedback edges, and branch labels.
2. Compare the matrix `direct_predecessors`, new facade imports, and typed predecessor stage. Reject numeric-order assumptions and any topology mutation.
3. Inspect each direct source and its own `.olean` timestamp. A root barrel, aggregate theorem, legacy object, or generated artifact is not direct evidence.
4. Recompute these dimensions separately: source/kernel, parity, mathematical closure, work, trust, compiled evidence, migration state, and blocker.
5. Treat a planned feature or route as unavailable even if a design document contains its proposed signature.

Audit nodes in dependency order. A downstream node cannot be closed while an exact parent on its path remains open. A typechecked facade may remain valuable as `migrated_open`; do not delete it merely to simplify status.

Audit production facades only under `examples/hypostructure_erdos_64_eg`; keep all cross-implementation theorems under the separate parity package.

## Audit each node packet

Rebuild the mandatory obligation table with a stable task ID for every local object, semantic bridge, branch, CT execution, trace, totality theorem, closure, and work result. Require each row to name an exact compiled producer or an exact missing producer.

For each node verify:

- the literal predecessor stage or `Core.Residual.Join` matches all paper parents;
- every inherited fact is read with a typed `Core.Residual.Query`/focus query from the one accumulated ledger;
- the output contains only the new paper-local payload;
- the application supplies only genuine `Core.Provision` primitives;
- Core/CT/Graph owns decisions, searches, outcomes, routes, ledger extension, coordinates, assembly, traces, and closure;
- every paper branch is represented and every residual has the exact declared consumer;
- `Core.Metadata` contains no manual obligation for a claimed complete endpoint; and
- no node-local replacement family, copied predecessor, custom handoff, status input, or caller-selected result exists.

If framework plumbing is duplicated in the application, use the framework-extension skill, add the generic capability and cross-domain fixture, and rewrite the node to the public executor before restoring status.

Classify any missing generic executor as a framework gap, never as permission for application-local routing.

## Audit mathematics, locality, and trust

- Compare the Lean predicates, quantifiers, constants, inequalities, branch scope, and conclusion with the exact responsibility in `original_erdos_64_proof.tex` in both directions.
- Reject sibling/later facts and facts whose producer depends on the node under review.
- Require a residual-owned finite universe or symbolic coverage theorem, exact visible checks, and a polynomial work proof. Reject raised resource limits and ambient graph/context enumeration.
- Require an actual `Core.Closure` result for every closed leaf; routes and parity are not closure.
- Run `#print axioms` on public endpoints and semantic theorems. Compare exact authored axioms with the current `trust-allowlist.json`, which may be empty.
- Run the production import checker and reject every legacy/parity import outside the parity package.

## Audit parity and evidence

Require theorem-level normalized parity for applicable public behavior: official input, branch partition, terminal/residual predicate, local facts, predecessor retention, work, and trust. Fixtures may pin deterministic codes but cannot replace the general parity theorem.

Do not copy an unsupported legacy input into the new API. Record deliberate manuscript-correct repairs and keep `math_status` independent from `parity_status`.

Require every non-missing Hypostructure obligation to cite a compiled new declaration. Reject stale source hashes, direct-node status inferred from an aggregate theorem, legacy evidence reused as new evidence, and web color used as an input to status.

Report reviewed nodes and edges, repaired owners, literal provenance, CT executions, all remaining residuals, direct kernel evidence, parity/math/work/trust judgments, matrix/evidence changes, and commands. A clean result has no unsupported closed state in the touched cone. If blocked, name the previous exact closed frontier and first missing producer; do not advance to the next node.
