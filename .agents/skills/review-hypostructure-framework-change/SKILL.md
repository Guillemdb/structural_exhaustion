# Review a Hypostructure Framework Change

Reconstruct the claimed consumer and verify that the change strengthens a reusable strategy rather than adding application glue.

Audit Core/domain ownership, predecessor identity, query provenance, branch witness retention, joins, target terminals, every remaining residual, early stopping, work composition, and the `ProblemDeclaration.ofDag` boundary.

Reject problem-specific constants or names in Core, detached payloads, manual routing, APIs with caller-supplied outcomes or finalizers, public compiled-chain construction, incomplete residual families, and success claims based only on compilation.

Require a focused Core fixture, a domain specialization, a real consumer migration, fresh builds, and axiom checks before PASS.
