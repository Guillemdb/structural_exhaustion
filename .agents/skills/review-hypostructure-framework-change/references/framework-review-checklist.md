# Hypostructure Framework Review Checklist

## Claim and status

- [ ] The claimed feature and owner are explicit.
- [ ] Specification, source, kernel, fixture, parity, math, work, and integration statuses are separate.
- [ ] Planned or baseline registry metadata is not presented as execution.

## Ownership and provision

- [ ] Every declaration passes the parameterization test.
- [ ] Every capability field is a legitimate author primitive.
- [ ] Framework outputs are generated internally.
- [ ] Public metadata contains no manual obligations.

## Residual execution

- [ ] The literal predecessor and stable residual are preserved.
- [ ] Existing data are read through typed queries.
- [ ] The framework performs the sole extension and all routing.
- [ ] Every outcome/residual has semantics and a consumer.
- [ ] Trace, totality, determinism, and polynomial work are proved.

## Transfer, trust, and evidence

- [ ] Public executors are exercised by appropriately independent fixtures.
- [ ] The production import firewall passes.
- [ ] Axiom output matches the reviewed allowlist and decision records.
- [ ] Migration rows were promoted only from fresh observed evidence.
- [ ] Focused and integration checks pass.
