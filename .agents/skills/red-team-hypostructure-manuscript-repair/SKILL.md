# Red-Team a Hypostructure Manuscript Repair

Assume the repair is wrong until every statement follows from the literal incoming stage and every outcome has an implemented, typed, terminating path.

## EG authority preflight

For any Erdős--Gyárfás Problem 64 task, read
`original_erdos_64_proof.tex` **FIRST**, before any API/process document,
migration row, generated view, living proof, or legacy Lean source. It is the
immutable sole authority for EG mathematics, strategy, node identity and
responsibility, and DAG topology. Freeze the exact quantified node contract,
branch alternatives, and exact incoming/outgoing DAG edges from that file.

Only after that contract and those edges are frozen may the matching
kernel-checked legacy `NodeX.lean` be read, and then only as implementation and
parity evidence. `proofs/erdos_64_eg/erdos_64_proof.tex` is a living,
non-binding editorial cross-check; it cannot repair, supplement, weaken,
strengthen, or redirect the original contract or edges. Any discrepancy blocks
the task: record it and stop rather than reconciling the sources or silently
changing the obligation. Never edit `original_erdos_64_proof.tex`.

After this preflight, API/process documents govern framework ownership,
capabilities, migration process, and status only. They never outrank or
reinterpret the original on EG mathematics, node responsibility, or DAG
topology.

## Establish independent evidence

Audit the actual repair sketch and code, not a summary. Read:

- the implicated manuscript theorem, diagrams, all predecessor paths, and all outgoing consumers;
- `HYPOSTRUCTURE_MIGRATION_GUIDE.md`, the relevant Core/Graph/PDE API sections, and matrix rows;
- the live Core, CT, Routes, Graph/PDE sources and fixtures used by the repair;
- [`references/red-team-checklist.md`](references/red-team-checklist.md); and
- the exact current trust allowlist and production import checker.

Read legacy source only when independently checking a parity claim. Never accept a legacy proof, artifact, or API as new-framework authority. Leave the source manuscript unchanged while findings remain.

## Reconstruct provenance

Rebuild the branch state from the exact root and direct predecessor stages. For every used value identify its original producer and typed ledger query. Reject later facts, sibling facts, copied predecessor fields, and any fact whose producer depends on the failed implication.

Distinguish the accumulated proof ledger from a mathematical closed-class ledger. Verify closed-class propagation removes prior classes before later quotient-compatible invariants; reject re-estimation disguised as a new node.

Compare the repair's incoming/outgoing edges with immutable manuscript topology. Require every side branch, join, feedback edge, and terminal, not only the preferred continuation.

## Attack quantifiers and residuals

Rewrite central statements literally and seek the smallest compatible countermodel. Test injection versus realization, pairwise versus simultaneous witnesses, swapped dependent quantifiers, representative existence, complete-context coverage, rank versus full product structure, and noncanonical existential choices.

Confirm that negating the failed claim creates a typed residual over the same incoming carrier. Automatically block a repair that weakens the theorem, assumes the missing result, renames it as an invariant, drops the negative branch, or defines a replacement family/image/subtype/enumeration of graphs, contexts, profiles, supports, witnesses, or realizations.

Run the both-sides test. Require measurable positive progress and a bounded/represented negative residual with a current consumer.

## Audit framework execution

For every Core/CT/domain step verify:

- its matrix row is not merely `planned`;
- the cited declaration exists in live source and elaborates through a permitted public import;
- author fields are irreducible `Core.Provision` primitives;
- searches, branch selection, target inputs, coordinates, assembly, routes, traces, ledger extension, and closures are framework outputs;
- the complete predecessor and all old queries survive the transition;
- every residual branch has an executable consumer or remains explicitly active; and
- complete metadata contains no manual obligation.

A manuscript CT chain is not proof of an executable route. Reject direct application calls to routing internals, custom handoffs, caller-authored outcomes, or application-built target inputs. If a generic operation is missing, require an ADR, correct owner, all residuals, Graph plus PDE/neutral fixtures, work, metadata, and trust before the repair can pass.

## Audit closure, termination, locality, and trust

Enumerate every leaf. Accept closure only through actual `Core.Closure` evidence: direct, strict progress, resource contradiction, exact imported contract, or acyclic reduction. For reuse/reduction, compute the consumer dependency cone independently and require strict routing-rank decrease; a green label or name match proves nothing.

Reject ambient graph/context/powerset/continuum enumeration, unbounded completion search, hidden exponential parameters, recursion without a strict measure, a certificate checker that repeats the global search, or raised resource limits. Require explicit residual-owned data, visible checks, and a polynomial work theorem.

Compare TeX and Lean hypotheses/conclusions literally. Search the dependency cone for admissions, unsafe declarations, authored axioms, compatibility/handoff names, forbidden imports, and problem-specific machinery in generic layers. Run `#print axioms` and permit authored external inputs only by exact current allowlist entry. An historical HSS or analytic exception is not authorized unless currently listed.

## Audit parity and status

Verify normalized semantic parity, not record-shape equality. A fixture is insufficient for a general branch equivalence. Ensure unsupported legacy assumptions are exposed as mathematical residuals, not copied into new capability fields.

Audit kernel, parity, mathematical closure, work, integration/evidence, trust, publication, and cutover independently. A fresh `.olean` proves elaboration only; parity and web status never close mathematics.

## Issue and iterate the verdict

Classify findings:

- **Blocking:** false mathematics, missing branch, unavailable API/route, hidden assumption, circularity, nonlocal/nonterminating computation, trust breach, or TeX--Lean mismatch.
- **Required cleanup:** copied state, wrong ownership, stale/incomplete evidence, missing fixture/work/metadata, or status conflation.
- **Advisory:** presentation only.

Return PASS only with no blocking or required-cleanup findings, all focused checks passing, and the source manuscript unchanged from the pre-sketch baseline. Otherwise record FAIL and the first exact obligation in the checklist, then return it to [`$repair-hypostructure-manuscript`](../repair-hypostructure-manuscript/SKILL.md). Repair the sketch/framework prototype and rerun the entire audit; never carry a finding as an assumption or ask the user to supply the missing lemma as the result.
