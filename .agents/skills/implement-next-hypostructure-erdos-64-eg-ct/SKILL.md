# Implement the Next Hypostructure Erdős 64 CT

Complete exactly one node/CT work packet, then stop. If an earlier scaffolded or typechecked node in the dependency cone lacks mathematical, parity, work, or trust evidence, finish that packet before creating a downstream facade.

## EG authority preflight

For any Erdős--Gyárfás Problem 64 task, read
`original_erdos_64_proof.tex` **FIRST**, before any API/process document,
migration row, generated view, living proof, or legacy Lean source. It is the
immutable sole authority for EG mathematics, strategy, node identity and
responsibility, and DAG topology. Freeze the exact quantified node contract,
branch alternatives, and exact incoming/outgoing DAG edges from that file.

Only after that contract and those edges are frozen may the matching
kernel-checked legacy `NodeX.lean` be read, and then only as implementation and
parity evidence. `proofs/erdos_64_eg/erdos_64_proof.tex` is a living,
non-binding editorial cross-check; it cannot repair, supplement, weaken,
strengthen, or redirect the original contract or edges. Any discrepancy blocks
the task: record it and stop rather than reconciling the sources or silently
changing the obligation. Never edit `original_erdos_64_proof.tex`.

After this preflight, API/process documents govern framework ownership,
capabilities, migration process, and status only. They never outrank or
reinterpret the original on EG mathematics, node responsibility, or DAG
topology.

## Read the binding node contract

After freezing the exact contract and edges, read
[`references/mandatory-node-template.md`](references/mandatory-node-template.md)
completely and fill it before editing Lean. Its topology, status,
literal-ledger, thin-application, framework-gap, locality, trust, and completion
gates are mandatory.

Then read:

- `HYPOSTRUCTURE_MIGRATION_GUIDE.md` sections 2, 6-8, 16.4, 17, 20, 23.6, 24, and 25;
- `migration/hypostructure/README.md` and the selected row of `migration/hypostructure/eg-node-matrix.csv`;
- the matching kernel-checked legacy node and framework declarations as implementation and parity evidence;
- the direct Hypostructure predecessors, `Problem.lean`, `InitialResidual.lean`, and the relevant public Core/Graph/CT/Routes sources;
- [`../design-hypostructure-proof/SKILL.md`](../design-hypostructure-proof/SKILL.md); and
- the selected `implement-hypostructure-ctN` skill.

Use legacy source to recover the checked implementation
strategy, define normalized parity observations, and inventory unsupported
premises. Never edit frozen legacy source and never import it from the
framework or new application; only `examples/hypostructure_parity` may import
both implementations.

## Select the one work packet

1. Inspect `git status --short` and preserve unrelated changes.
2. Read the manuscript DAG and matrix `direct_predecessors`; do not use numeric order as topology.
3. Recompute the status basis from the direct new source and its own fresh `.olean`. Do not use a root barrel, aggregate theorem, generated color, or legacy `.olean` as node evidence.
4. Keep `new_kernel`, `parity_status`, `math_status`, `work_status`, `web_evidence`, migration `status`, and `blocker` independent.
5. Select the earliest exact paper path whose direct parents are `migrated_closed`, or select the earliest existing facade that must be completed before that condition can hold. A root node is the only predecessor-free exception.
6. Freeze every manuscript node, edge, branch label, join, and exit. Do not modify diagram topology or invent an internal node to bypass a paper edge.

Do not interpret `typechecked` as `migrated_closed`. Do not advance because parity passes. An open translated residual is honest `migrated_open` evidence, not a green proof node.

## Verify every required API

For each `required_features` and `ct_ids` entry:

1. inspect the API matrix status;
2. locate the declaration in live Hypostructure source;
3. verify the smallest public import permitted by the firewall;
4. run a focused elaboration check when freshness or the exact surface is uncertain; and
5. inspect `Routes.Registry`/`Routes.Accumulated` for every cross-CT edge.

A `planned` matrix row or design signature is not callable. Treat inability to express the thin node as a framework gap: stop application work and use [`../extend-hypostructure-framework/SKILL.md`](../extend-hypostructure-framework/SKILL.md). State the missing operation without Erdős names, choose Core/CT/Routes/Graph ownership, add an ADR and Graph plus PDE/neutral fixtures, then resume only through the public executor. Leave the node open if that reusable slice cannot be completed in this packet.

## Implement one thin node

Use the literal stage returned by the direct predecessor, or a framework-owned `Core.Residual.Join` for an existing manuscript join. Only node 1 may call `Core.Residual.Ledger.initial`.

Implement in this order:

1. Define only fixed Problem-64 data, predicates, deciders, and semantic facts first introduced by this paper node.
2. Retrieve every inherited value through one compositional `Core.Residual.Query`, `Focus.ActiveQuery`, or public CT/domain query. Use `preserve`, `map`, and `and`; never copy an earlier payload.
3. Supply only the minimal `Capability`, certificate, or domain adapter identified by `Core.Provision`.
4. Invoke the public Core, Graph, or CT automation on the literal predecessor. Do not pattern-match merely to choose a route that the framework should own.
5. Retain the framework's exact stage, decision branches, terminal-indexed outcome, typed trace, semantics, totality, and work evidence.
6. Prove every paper-local branch and outgoing payload, but no successor responsibility.
7. Add complete `Core.Metadata` when that exporter surface is available; never represent unfinished mathematics as a callable `ManualObligation` or application input.

Reject application-owned route constructors, output ledgers, custom handoffs, restaging, replacement residuals, copied predecessor fields, caller-selected terminals, assumed closure, manual coordinate/gluing outputs, detached occurrence families, or a new family of graphs/contexts/witnesses not owned by the incoming residual.

## Prove locality, closure, and trust

- Scan only an explicit finite family already owned by the incoming residual. Use symbolic coverage for unbounded outside contexts.
- Record visible checks and a `Core.PolynomialCheckBudget`; compare the asymptotic bound with the frozen legacy baseline when one exists.
- Close a branch only through the exact `Core.Closure` evidence. A route, parity theorem, node name, or dashboard state is not closure.
- Run `#print axioms` on the public endpoint and new semantic theorems. Accept authored external axioms only when their exact path and declaration appear in `migration/hypostructure/trust-allowlist.json`; the current allowlist, not historical policy, is authoritative.
- Never raise heartbeat, recursion, timeout, or memory limits to make a nonlocal design compile.

## Prove parity without importing legacy production

Add or update a module under `examples/hypostructure_parity` only after the new node builds independently. Compare normalized manuscript-visible input, branch partition, terminal/residual predicate, public local facts, predecessor retention, work, and trust. Do not compare private record shapes or copy legacy outputs.

If the legacy node has an unsupported premise or wrong branch, document the deliberate difference and keep `math_status` honest. Parity can pass while mathematical closure remains open; mathematical closure can be correct before parity is available.
