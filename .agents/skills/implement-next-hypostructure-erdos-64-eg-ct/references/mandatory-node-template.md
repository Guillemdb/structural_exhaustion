# Mandatory Hypostructure Erdős Node Contract

Fill this contract before editing, reviewing, or promoting one Erdős node. Keep it with the work packet or reproduce every field in the implementation record.

## 1. Status and authority snapshot

| Field | Required observation |
|---|---|
| Repository revision/worktree | Commit plus relevant dirty files |
| Matrix row | Exact row from `eg-node-matrix.csv` |
| New source and direct `.olean` | Path, present/missing, fresh/stale |
| Required API rows | Feature IDs and current statuses |
| Parity | `not_run`, checked result, or exact blocker |
| Mathematics | closed/open and exact paper obligation |
| Work | captured/not captured and theorem |
| Trust | transitive axiom result and allowlist entries |
| Evidence | compiled Hypostructure declaration/metadata only |

Treat specifications as intended contracts, live source as declaration truth, and fresh object files as kernel evidence. A planned row is unavailable. A legacy object, parity theorem, aggregate import, or generated color cannot establish any new-framework field.

## 2. Freeze paper identity and topology

Read the exact diagram, definitions, and proof in the immutable mathematical
authority `original_erdos_64_proof.tex`, then inspect the corresponding
kernel-checked legacy node as implementation/parity evidence, and fill. The
living manuscript is not an obligation source.

| Field | Required value |
|---|---|
| Node ID | One existing manuscript node |
| Direct incoming edges | Every exact producer and branch label |
| Direct outgoing edges | Every exact consumer/terminal and branch label |
| Literal incoming stage | Direct predecessor facade or Core join |
| Inherited facts | Fact, original producer, and typed query |
| Local responsibility | Only the mathematical move at this node |
| New payload | Only data/proposition first proved here |
| CT/Core/domain executor | Current public declaration and import |
| Complementary residuals | Exact paper edge and typed consumer |

Reject any edit that adds, removes, renames, splits, merges, or reorders a paper node, edge, branch, join, or exit. Numeric node order is not dependency order.

Include locally introduced objects, predicate reflection, every branch, executor result, trace, soundness, totality, closure evidence, and work theorem. Do not assign a successor's theorem or sibling fact to this node.

## 4. Enforce the only legal state flow

```text
exact predecessor stage or typed Core join
  -> one accumulated proof ledger
  -> typed queries for inherited facts
  -> one new local semantic certificate
  -> public framework/domain executor
  -> framework-owned branch, trace, route, and ledger extension
  -> exact paper successor stage(s)
```

Only node 1 may seed `Core.Residual.Ledger.initial`. Every later node consumes the literal predecessor result and its one accumulated ledger. Use the live forms of:

- `Core.Residual.Query` with `residual`, `latest`, `preserve`, `map`, and `and`;
- `Core.Residual.StageNode.derive` for a data-bearing successor;
- `Core.Residual.Decision` and `Core.Residual.Focus` for existing paper decisions;
- `Core.Residual.Join.execute` for an existing multi-parent paper join; and
- public CT `Automation` or Graph executors for CT steps.

Do not seed, clone, checkpoint, restage, or manually extend a ledger. A branch decision keeps sibling evidence in its literal predecessor tree. Do not convert it into an application sum or parallel branch ledger.

## 5. Audit author primitives

| Input | `Core.Provision` role | Why irreducible | Already in residual/query? |
|---|---|---|---|
| | | | |

The application may supply fixed Problem-64 constants, current residual predicates, local deciders, finite data already owned by the residual, and exact semantic theorems. Record every framework output separately: the framework must generate search results, selected terminals, residual constructors, traces, routes, coordinate actions, ledger updates, closure tags, and composed work.

Reject a capability field that lets the caller provide the fact, outcome, route, target input, closure, or downstream payload that the node is meant to derive.

## 6. Keep output thin

Do not put any of the following in a node-local output:

- predecessor, ledger, root context, or inherited fact;
- an equality back to the predecessor;
- caller-selected branch, terminal, closure, or feasibility result;
- output-ledger alias, custom route, custom handoff, or checkpoint;
- manually transformed/glued graph;
- a new graph, context, support, witness, realization, image, subtype, or chosen-representative family replacing data in the incoming residual; or
- dashboard, migration, parity, or publication status.

If a required view of the literal residual is missing, add a generic Core query/projection. If execution is missing, follow the framework-gap protocol; never create application plumbing.

## 7. Gate current capability and routing

For each intended API, record:

| Feature/route | Matrix status | Source declaration | Public import | Focused check | Usable? |
|---|---|---|---|---|---|
| | | | | | |

Do not call a proposed signature or `planned` route. A manuscript CT chain does not make its transitions executable. Every outcome must remain typed. Every cross-CT edge must use a framework executor that consumes the complete source stage, performs semantic discovery, invokes the target public executor, records provenance, preserves all source queries, and returns the target-labelled accumulated stage.

For a gap, stop the node and create a generic work item: vocabulary-neutral statement, owner, primitive inputs, generated outputs, all residual branches, Graph plus PDE/neutral interpretations, ADR, fixtures, work, metadata, and trust audit. Resume only after it compiles.

## 8. Preserve production/parity isolation

- Production framework: import only Mathlib and allowed `Hypostructure` layers.
- New Erdős application: import only Mathlib, public Hypostructure modules, and its own direct predecessors.
- Parity package: the only place allowed to import both new and frozen legacy packages.
- Legacy source: read-only behavioral evidence; never a producer in the new proof.

Compare normalized paper-visible behavior, not record layouts. Record unsupported legacy premises as parity differences and explicit mathematical residuals; never reproduce them as new assumptions.

## 9. Locality, closure, and trust

- Name the residual-owned finite family or symbolic coverage theorem.
- State exact visible checks and a `Core.PolynomialCheckBudget`.
- Use only structural recursion with an explicit decreasing measure.
- Forbid ambient graph, subgraph, coloring, context, powerset, or completion enumeration.
- Close only through `Core.Closure`: direct evidence, strict progress, resource contradiction, exact imported contract, or acyclic reduction.
- Run `#print axioms` on the endpoint and semantic theorems.
- Accept authored axioms only by exact current `trust-allowlist.json` path and declaration; do not assume a historical exception remains authorized.

## 10. Independent completion gates

Mark each gate separately:

- **Kernel:** direct source has a fresh direct `.olean`.
- **Parity:** normalized old/new observations are proved or a deliberate difference is recorded.
- **Mathematics:** the exact local paper responsibility is proved and every complementary residual has a consumer or honest active-frontier status.
- **Work:** the local bound is proved and any regression is justified.
- **Trust:** transitive axioms match the current allowlist.
- **Evidence:** compiled Hypostructure metadata/web evidence points to the direct declaration.

Set `migrated_closed` only when every required gate for closure passes. Use `migrated_open` for a kernel-checked faithful translation with a named mathematical residual. Never promote from `typechecked` or `parity_checked` by inference.

## 11. Completion report

Record the exact node and edges, predecessor stage, ledger queries, new payload, public executor, residual consumers, API availability evidence, focused/wide commands, direct kernel result, parity theorem, mathematical verdict, work bound, trust delta, matrix changes, and next topology-ready node. Stop after this packet.
