\subsection{CT9: Exact label-fibre overload}
\label{sec:ct9-automation}

CT9 records a nullary partition audit boundary and evaluates the exact label-fibre view from the original input during overload search. Labels are searched in enumeration order for the first capacity violation. The two terminals contain either the overload witness or a certificate that every label fibre is bounded.

A theorem instantiation supplies the problem-specific static entries in the first table. The framework constructs every remaining node output, exhaustive decision, certificate, residual, edge, and trace theorem.

\texttt{StructuralExhaustion.CT9.run} returns the evidence-carrying \texttt{ExecutionResult}. \texttt{ct9\_\allowbreak{}execute}, \texttt{ct9}, and \texttt{ct9\_\allowbreak{}total} expose execution, semantic soundness, and totality, respectively.
