\subsection{CT8: Finite repetition and response}
\label{sec:ct8-automation}

CT8 searches the input sequence for the first ordered pair with repeated exact type. Absence gives the no-repetition certificate. A repeated pair is passed to exhaustive response comparison, which returns either a separating response context or the verified smaller-object removal residual.

A theorem instantiation supplies the problem-specific static entries in the first table. The framework constructs every remaining node output, exhaustive decision, certificate, residual, edge, and trace theorem.

\paragraph{Public automation surface.}
\texttt{StructuralExhaustion.CT8.run} returns the evidence-carrying \texttt{ExecutionResult}. \texttt{ct8\_\allowbreak{}execute}, \texttt{ct8}, and \texttt{ct8\_\allowbreak{}total} expose execution, semantic soundness, and totality, respectively.
