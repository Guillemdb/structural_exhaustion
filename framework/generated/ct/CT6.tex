\subsection{CT6: Ordered activity failure}
\label{sec:ct6-automation}

CT6 performs one ordered first-failure search. A hit records the failing index, the clean prefix, and its structural failure datum; exhaustive absence records the no-failure theorem together with the computed active total. Both branches are semantic residuals with the original branch context.

A theorem instantiation supplies the problem-specific static entries in the first table. The framework constructs every remaining node output, exhaustive decision, certificate, residual, edge, and trace theorem.
