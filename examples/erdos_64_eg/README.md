# Erdős Problem 64 framework example

This independent Lake package exercises the reusable structural-exhaustion
framework on the first reductions of Erdős Problem 64. It depends on Mathlib
4.31.0 and on the framework through `path = "../../lean"`. Framework modules
remain application-independent.

The public proposition in `Erdos64EG/OfficialStatement.lean` is the exact
right-hand side of `Erdos64.erdos_64` in Google DeepMind's
formal-conjectures repository, pinned at commit
`bcaee6031a085e19432540650e1039b7fd1cea36`. It remains in Mathlib
`SimpleGraph` vocabulary. Internal CT adapters carry a proved bridge to that
statement.

The internal ambient type is the framework's `Graph.FiniteObject`: the same
Mathlib `SimpleGraph` together with the explicit `FinEnum`/adjacency data
needed by deterministic search.

The verified proof slice contains:

- a proved equivalence between the finite executable length predicate and
  the official `∃ k ≥ 2, length = 2^k` predicate;
- CT1 generated by the framework graph adapter, including exhaustive bounded
  Mathlib-walk support codes and exact encode/decode proofs;
- CT2 over Mathlib `SimpleGraph.Dart`, with single-edge deletion, exact
  edge-count decrease, baseline preservation, target transport by
  `Walk.mapLe`, and the resulting degree-three endpoint theorem;
- the CT3 author contract for the manuscript's boundaried-compression stage:
  pieces, compatible contexts, candidates, rows, `X ⊕_T Y`, structural
  admissibility, strict lexicographic decrease, finite domains, and primitive
  deciders are mapped directly into the generic CT3 engine;
- exact CT3 response bits computed by the existing CT1 target decider, with a
  proved bridge from response equality to target equivalence in every declared
  context;
- exported CT3 soundness, typed-trace, and totality theorems for every concrete
  value of the manuscript contract;
- an executable `K₄` smoke fixture for the problem-specific CT1 runner.

The remaining manuscript sections extend this verified proof slice with their
corresponding closure-tactic contracts.

From the repository root:

```bash
make mathlib-cache
make erdos-example-build
```

Or directly:

```bash
cd examples/erdos_64_eg
lake exe cache get
lake build
```
