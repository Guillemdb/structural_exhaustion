\subsection{CT17 --- Target thickening}\label{ct:17}\label{target-thickening-bounded-offsets-against-sparse-targets}
\ctsignature{\(\begin{array}{@{}l@{}}P_{1\to17}\to \ctcert{C1,C5};\\
\ctint{P_{17\to8},P_{17\to14}};\\
\ctrec{P_{17\to3},P_{17\to10}}.\end{array}\)}

CT17 thickens a sparse target by offsets realized in one branch state.  Its
entry fixes the target set, offset family, completion classes, arithmetic
datum, branch state, and C1 and C5 claims.  Scope certifies that a bounded
offset family with finite arithmetic leverage is available.

Compatibility is tested before block construction.  Incompatible offsets
enter a separation node and construct an exact CT3 type input or CT10 label
input; they cannot be pooled in a target block.  Compatible offsets pass
through the one-successor block certificate and then the scale decision.  A
finite-range state enters the survivor decision, which closes by C5 or
constructs an exact CT8 input.  A repeated-increment state enters the
arithmetic decision, which closes by C1 or constructs an exact CT14 input.

The two scale branches have different indexed states and different consumer
payloads.  Consequently a finite survivor list cannot be used as aggregate
mass, and a large-scale arithmetic residual cannot be presented to the
finite-state pumping tactic.

\begin{itemize}
\tightlist
\item \textbf{S-Def} fixes target, offsets, completion classes, and arithmetic data.
\item \textbf{S-Dich} is split among compatibility, scale, survivor, and arithmetic decisions.
\item \textbf{S-Comp} is carried by block construction and the C1/C5 certificates.
\item \textbf{S-Rout} and \textbf{S-Trig} are exact CT3, CT8, CT10, and CT14 inputs.
\end{itemize}
