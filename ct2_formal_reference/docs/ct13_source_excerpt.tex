\subsection{CT13 --- Tiered charging}\label{ct:13}\label{tiered-charging-scheme-failure-as-a-demand}
\ctsignature{\(P_{\ast\to13}\to \ctcert{C4};\ \ctint{P_{13\to4},P_{13\to9},P_{13\to14}};\ \ctrec{\ctnone}.\)}

CT13 is an exact two-tier accounting machine.  Its entry fixes the tier
account, availability datum, measurable resource, branch state, and C4 claim.
The executable framework separates tier-one availability, canonical payment,
minimal fallback obstruction, canonical tier-two payment, resource
reconciliation, and the combined capacity comparison.

The availability node produces either an \texttt{AvailableState} or an
\texttt{UnavailableState}.  The available branch must pass through tier-one
payment certification and a separate CT4 payload-construction node.  The
unavailable branch must pass through fallback certification before the
reconciliation decision.  A reconciled state proceeds through the one-successor
combined comparison and closes by C4.  An overlap state proceeds to a separate
routing decision and constructs an exact CT9 fibre input or CT14 mass input.

This flow prevents an availability failure from being treated as a free
fallback choice.  The fallback state contains both minimality and canonicality,
and no overlap route exists until the reconciliation node has produced an
\texttt{OverlapState}.

\begin{itemize}
\tightlist
\item \textbf{S-Det} is split between tier-one payment and minimal canonical fallback.
\item \textbf{S-Dich} is the availability decision and the reconciliation decision.
\item \textbf{S-Comp} is the resource-reconciliation and combined-capacity certificate.
\item \textbf{S-Rout} and \textbf{S-Trig} are exact CT4, CT9, and CT14 inputs.
\end{itemize}
